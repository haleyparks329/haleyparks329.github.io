# Website Redesign Handoff

## Folder Structure

```text
docs/
└── design/
    ├── WEBSITE_VISION.md
    ├── WEBSITE_REDESIGN_PLAN.md
    ├── WEBSITE_IMPLEMENTATION.md
    └── FUTURE_IDEAS.md
```

## Workflow

### Step 1

Read `WEBSITE_VISION.md`.

Do not code.

Explain your understanding of the philosophy.

STOP.

### Step 2

Read `WEBSITE_REDESIGN_PLAN.md`.

Audit the existing site and map the redesign to the current codebase.

Do not code.

STOP.

### Step 3

Read `WEBSITE_IMPLEMENTATION.md`.

Produce a concrete implementation plan by phase.

Do not code.

STOP.

### Step 4

Implement **one phase only**.

After every phase:

- Run tests
- Run production build
- Verify responsive layouts
- Report files changed
- Commit once

STOP and wait for review.

### Final

Read `FUTURE_IDEAS.md`.

Do **not** implement it.

Instead review whether the new architecture naturally supports those ideas.
