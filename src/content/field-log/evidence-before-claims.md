## What changed

This note set a public writing rule for AI work: implemented artifacts, source data, and caveats should carry more weight than large promises.

## Why

The projects touch personal data, career reasoning, and agent workflows. Those areas become less trustworthy when public pages blur what exists with what is planned.

## What I noticed

The interesting work can still be ambitious without pretending every prototype is a finished product. Clear boundaries make the work easier to trust, not smaller.

## Next

Keep project pages grounded in repositories, demos, validation results, and explicit limitations.

## Boundary

This does not mean every project needs to be complete before it can be public. It means the public version should make maturity visible.
